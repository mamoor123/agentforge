## 🤔 What is AgentForge?

> **Stop hunting across 50 websites. Find every AI agent in one place.**

AgentForge is the **#1 curated marketplace** for AI agents. We've cataloged **1000 real, verified AI tools** across **22 categories** — from Marketing to Coding, Creative to Cybersecurity — so you can discover, compare, and deploy the perfect AI agent for any task.

**Think Product Hunt meets G2 for AI agents.**

## 🚀 Quick Start

### Get running in under 60 seconds

<br />

```bash
# 1️⃣ Clone the repo
git clone https://github.com/mamoor123/agentforge.git
cd agentforge/agentforge

# 2️⃣ Install dependencies
npm install

# 3️⃣ Copy env template (optional — for auth & reviews)
cp .env.example .env.local
# Fill in your Supabase URL + anon key

# 4️⃣ Start development server
npm run dev
```

<br />

Open **[http://localhost:3000](http://localhost:3000)** and start exploring 🎉

## 🌐 Deploy to Vercel

### One-click deploy — zero configuration

<br />

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/mamoor123/agentforge)

<br />

Or use the CLI:

```bash
# Install Vercel CLI
npm i -g vercel

# Deploy
vercel

# Deploy to production
vercel --prod
```

<br />

> **Note:** Auth, reviews, and favorites require Supabase. Set `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` in your Vercel environment variables. Run the SQL migration from `supabase/migrations/001_initial_schema.sql` in your Supabase SQL editor.

## 📁 Project Structure

```
agentforge/
├── src/

│   │   ├── page.tsx                    # 🏠 Landing page
```

## 🤝 Contributing

### We'd love your help making AgentForge better!

<br />

#### Option 1: Submit an Agent 🚀

[Open an issue](https://github.com/mamoor123/agentforge/issues/new) with:
- Agent name
- Website URL
- Category
- Brief description

<br />

#### Option 2: Submit a PR 🔧

```bash
# 1. Fork the repo
# 2. Create your branch
git checkout -b add-new-agent

# 3. Add your agent to src/data/agents.json
# 4. Commit your changes
git commit -m "Add [Agent Name] to [Category]"

# 5. Push and create PR
git push origin add-new-agent
```

<br />

#### Agent Data Structure

```json
{
  "id": "your-agent",
  "name": "Your Agent",
  "tagline": "One-line description",
  "description": "Longer description...",
  "category": "Marketing",
  "categorySlug": "marketing",
  "icon": "🚀",
  "rating": 4.5,
  "users": 100000,
  "price": "Freemium",
  "featured": false,
  "new": true,
  "tags": ["tag1", "tag2", "tag3"],
  "creator": "Your Company",

  "features": ["Feature 1", "Feature 2"]
```
